# Zitadel — Stacker Deploy HOWTO

Three containers: zitadel-api (ghcr.io/zitadel/zitadel:v4.16.0) + postgres:17.10-alpine + redis:7.4.9-alpine.

## Setup

```bash
cp .env.example .env && ./scripts/generate-secrets.sh
stacker config setup server --ip <SERVER_IP> --user root --key ~/.ssh/id_ed25519
```

## Deploy

```bash
stacker deploy --target server --force-rebuild
```

## Post-deploy

```bash
# Reset stale DB on re-deploy
docker compose -p project down -v
docker compose -p project up -d
```

## Access

Open `http://<SERVER_IP>:8080/ui/login` in a browser.

Default credentials (generated by `start-from-init`):
- Username: `zitadel-admin@zitadel.localhost`
- Password: `Password1!`

## Notes

- Port: 8080 (maps to container port 8080)
- Uses legacy login UI (LOGINV2_REQUIRED=false) since stacker strips service labels needed for Traefik routing
- Named volumes: postgres-data
- Change `ZITADEL_MASTERKEY` and `POSTGRES_ADMIN_PASSWORD` before exposing to internet
- Database credentials: postgres / POSTGRES_ADMIN_PASSWORD
